# 目录结构建议草稿

> 记录对当前项目结构的思考和改进建议，仅供参考，不强制执行。

## 当前结构

```
dssgym/
├── dssgym/              # 核心环境代码
│   ├── env.py
│   ├── env_register.py
│   ├── circuit.py
│   └── ...
├── ev_demand/           # EV需求数据 + 生成脚本
├── systems/             # OpenDSS电力系统算例
├── results/             # 训练/测试输出
├── ppo_agent.py         # PPO入口
├── rules_agent.py       # 规则策略入口
├── reward_curve.py      # 奖励曲线工具
└── test_results_analysis.py
```

## 改进空间

### 1. 入口脚本集中管理

根目录有4个入口脚本，可考虑收入独立目录如 `scripts/` 或 `agents/`：
- `ppo_agent.py`
- `rules_agent.py`
- `reward_curve.py`
- `test_results_analysis.py`

**注意**：需评估是否影响现有工作流。

### 2. 数据目录规范化

- `ev_demand/` 混合了 `.csv` 数据和 `.py` 生成脚本
- `systems/` 混合了 Python 工具脚本和算例数据

可考虑分离为：
```
data/
├── ev_demand/           # 仅CSV
└── systems/             # 仅DSS文件
scripts/
├── ev_demand_gen.py     # 移入
└── convert_freq.py      # 移入
```

**注意**：需考虑现有路径引用兼容性。

### 3. 测试目录缺失

当前没有 `tests/` 目录，可考虑添加。

---

## 总结

以上仅为初步想法，考虑到本项目的历史包袱（基于 powergym 二次开发、已有大量实验产出），任何结构性改动都需谨慎评估兼容性。
