# Agent Instructions

This repository uses a layered agent-instruction model. Read the following files in order before acting:

1. [AGENT_RULES.md](./AGENT_RULES.md)
2. [AGENT_CONTEXT.md](./AGENT_CONTEXT.md)
3. [./.codex/Agent.md](./.codex/Agent.md)

## Priority

- `AGENT_RULES.md` has the highest priority for execution rules.
- `AGENT_CONTEXT.md` is the source of truth for project background and runtime facts.
- `.codex/Agent.md` is supplemental context generated by tools. If it conflicts with root-level docs, root-level docs win.

## Conflict Handling

- If `AGENT_RULES.md` and `AGENT_CONTEXT.md` conflict, follow `AGENT_RULES.md`.
- If root-level docs conflict with `.codex/Agent.md`, follow the root-level docs.
- If a rule or fact is still ambiguous after reading all files, stop and ask the developer for confirmation before proceeding.

## Maintenance

- After ambiguity is resolved by the developer, add the clarification back into the appropriate root-level document.
- Keep `AGENT_RULES.md` focused on stable, project-agnostic execution constraints.
- Keep `AGENT_CONTEXT.md` focused on project-specific background, codebase landmarks, runtime truths, and concrete workflows.
